## Content

Default HikariCP pool size for payment-adjacent services should scale with expected job concurrency, not request concurrency. Nightly batch jobs (exports, reconciliation) can exhaust a request-sized pool even when live traffic is low.

Rule of thumb: pool size >= max concurrent batch workers + 20% headroom. Always set a connectionTimeout so callers fail fast instead of queuing indefinitely.
